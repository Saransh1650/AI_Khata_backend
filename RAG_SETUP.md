# AI Khata — RAG Memory System Setup Guide\n\n## Overview\n\nThis guide explains how to set up and use the RAG (Retrieval Augmented Generation) memory system that transforms AI Khata from an \"Inventory Manager\" to a \"Sales Intelligence Partner\".\n\n## What is RAG Memory in AI Khata?\n\nThe RAG memory system learns from your shop's historical behavior and provides experience-driven guidance rather than reactive inventory alerts.\n\n**Philosophy Shift:**\n- **Old**: \"Stock is low → Buy more\"\n- **New**: \"Based on how your shop behaves, these products are likely to sell next or increase revenue\"\n\n## Setup Instructions\n\n### 1. Database Setup\n\nFirst, run the RAG memory schema to create the necessary tables:\n\n`bash\npsql $DATABASE_URL -f src/config/rag_memory.sql\n`\n\nThis creates tables for:\n- `shop_memory` - Learned behavioral patterns\n- `product_relationships` - Product association intelligence\n- `experience_insights` - High-level shop insights\n- `memory_usage` - Memory effectiveness tracking\n\n### 2. Initialize Store Memory (One-time per store)\n\nFor existing stores with transaction history, initialize the RAG memory:\n\n`bash\ncurl -X POST http://localhost:3000/memory/initialize/{storeId} \\\n  -H \"Authorization: Bearer {token}\" \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\"lookbackDays\": 90}'\n`\n\nThis will:\n- Learn from the last 90 days of transactions\n- Discover product relationships\n- Generate initial experience insights\n\n### 3. Verify Setup\n\nCheck memory health:\n\n`bash\ncurl -X GET http://localhost:3000/memory/health/{storeId} \\\n  -H \"Authorization: Bearer {token}\"\n`\n\n## How RAG Memory Works\n\n### Automatic Learning\n\nThe system automatically learns from new transactions when:\n- New ledger entries are created\n- Bills are processed with line items\n- Background workers run (every time insights refresh)\n\n### Memory Types\n\n1. **Product Behavior Memory**\n - How specific products perform in your shop\n - Average quantities, pricing patterns\n - Day-of-week preferences\n\n2. **Product Relationship Memory** (MOST IMPORTANT)\n - Which products are bought together\n - Sequential purchase patterns\n - Complementary product intelligence\n\n3. **Seasonal Pattern Memory**\n - How shop responds to festivals/seasons\n - Festival-specific product pairings\n\n4. **Operational Rhythm Memory**\n - Peak business days and times\n - Transaction value patterns\n\n## API Endpoints\n\n### Get RAG-Driven Recommendations\n`\nGET /memory/recommendations/{storeId}\n`\nReturns strength-based suggestions and expansion opportunities.\n\n### Get Sales Expansion Guidance\n`\nGET /memory/expansion/{storeId}\n`\nReturns cross-sell opportunities and missing complementary products.\n\n### Get Product Experience\n`\nGET /memory/products/{storeId}?products=milk,bread,eggs\n`\nReturns learned behavioral patterns for specific products.\n\n### Get Product Relationships\n`\nGET /memory/relationships/{storeId}?minStrength=0.5&type=frequently_together\n`\nReturns product association intelligence.\n\n### Get Experience Insights\n`\nGET /memory/insights/{storeId}\n`\nReturns high-level shop intelligence and patterns.\n\n## Integration with Existing AI\n\nThe RAG system integrates seamlessly with existing AI insights:\n\n### Priority Order (as per specification)\n1. **RAG Memory** (Historical Shop Understanding) ← **Highest Priority**\n2. Current shop context (recent activity)\n3. Inventory state\n4. External context (festival timing)\n\n### Automatic Switching\n- **10+ transactions**: Uses traditional AI guidance\n- **10+ transactions**: Starts building RAG memory\n- **20+ transactions**: Switches to RAG-driven guidance\n- **50+ transactions**: Full experience-driven intelligence\n\n## Understanding the Guidance\n\n### New Guidance Types\n\n1. **strength_amplification**\n - Products that historically perform well\n - Memory-validated recommendations\n\n2. **sales_expansion**\n - Cross-sell opportunities from learned patterns\n - Missing complementary products\n\n3. **momentum_pattern**\n - Current trends validated against shop memory\n - Experience-guided pattern recognition\n\n4. **festival_experience** or **festival_preparation**\n - Festival guidance based on shop's historical response\n - Memory-validated seasonal recommendations\n\n5. **shop_intelligence**\n - High-level insights about shop behavior\n - Business momentum analysis\n\n### Example RAG Response\n\n`json\n{\n  \"mode\": \"EXPERIENCE_NORMAL\",\n  \"philosophy\": \"experience_driven\",\n  \"guidance\": [\n    {\n      \"type\": \"strength_amplification\",\n      \"insight\": \"Based on your shop's proven track record\",\n      \"items\": [\n        {\n          \"product\": \"Milk\",\n          \"status\": \"GOOD\",\n          \"reason\": \"Consistently sells in high volumes\",\n          \"action\": \"Monitor closely - key revenue driver\",\n          \"memoryStrength\": 0.85,\n          \"priority\": \"strength_product\"\n        }\n      ]\n    },\n    {\n      \"type\": \"sales_expansion\",\n      \"insight\": \"Revenue growth opportunities from your customer patterns\",\n      \"crossSell\": [\n        {\n          \"trigger\": \"Bread\",\n          \"suggest\": \"Butter\",\n          \"reason\": \"78% of customers buying Bread also want Butter\",\n          \"strength\": 0.78,\n          \"action\": \"Display together or suggest during sale\"\n        }\n      ],\n      \"missingItems\": [\n        {\n          \"existing\": \"Tea\",\n          \"missing\": \"Biscuits\",\n          \"opportunity\": \"Customers buying Tea often want Biscuits\",\n          \"action\": \"Consider stocking Biscuits to increase basket size\"\n        }\n      ]\n    }\n  ]\n}\n`\n\n## Memory Health Monitoring\n\n### Health Status Levels\n- **Weak**: < 5 memory entries (need more data)\n- **Developing**: 5-10 entries (building patterns)\n- **Good**: 10-20 entries + relationships (useful intelligence)\n- **Excellent**: 20+ entries + 10+ relationships + insights (full intelligence)\n\n### Maintenance\n\n**Clean up old memory** (removes low-confidence old patterns):\n`bash\ncurl -X POST http://localhost:3000/memory/cleanup/{storeId} \\\n  -H \"Authorization: Bearer {token}\" \\\n  -d '{\"maxAge\": 180, \"minConfidence\": 0.20}'\n`\n\n**Manual learning trigger**:\n`bash\ncurl -X POST http://localhost:3000/memory/learn/{storeId} \\\n  -H \"Authorization: Bearer {token}\" \\\n  -d '{\"days\": 30}'\n`\n\n## Key Benefits\n\n### For Shop Owners\n1. **Proactive Intelligence**: \"What should I stock?\" instead of \"What am I low on?\"\n2. **Revenue Focus**: Suggestions to increase basket size and sales\n3. **Personalized Advice**: Based on YOUR shop's behavior, not generic patterns\n4. **Experience Continuity**: AI remembers and learns from your success patterns\n\n### For the System\n1. **Reduces AI Hallucinations**: Grounded in actual shop data\n2. **Improves Over Time**: More transactions = better intelligence\n3. **Context-Aware**: Festival suggestions based on shop's historical response\n4. **Relationship-Driven**: Focus on product combinations that drive sales\n\n## Troubleshooting\n\n### \"No RAG guidance generated\"\n- **Cause**: Less than 10 transactions in the system\n- **Solution**: System will use traditional AI until more data is available\n\n### \"Memory health is weak\"\n- **Cause**: Insufficient transaction variety or volume\n- **Solution**: Keep adding diverse transactions; system learns over time\n\n### \"No product relationships found\"\n- **Cause**: Transactions don't have multiple line items\n- **Solution**: Ensure bills are processed with complete line item details\n\n### \"Experience insights empty\"\n- **Cause**: Need more transaction history (< 20 transactions)\n- **Solution**: System generates insights automatically as data grows\n\n## Migration Notes\n\n- **Existing Data**: Run initialization for stores with existing transaction history\n- **New Stores**: RAG memory builds automatically as transactions are added\n- **Backward Compatibility**: Traditional AI guidance still works for new/low-data stores\n- **Performance**: RAG queries are optimized and cached; no significant performance impact\n\n## Philosophy in Action\n\n**Example Scenario**: Diwali is approaching\n\n**Traditional System**:\n\"Diwali is coming. Stock these generic items: sweets, oil, dry fruits\"\n\n**RAG System**:\n\"Based on your shop's Diwali history: Your customers always buy Milk + Sugar together during Diwali (89% correlation). You also sell 3x more Ghee than usual. Consider stocking extra Milk and Ghee - they're proven revenue drivers during your Diwali seasons.\"\n\nThe difference: **Generic advice vs. Shop-specific intelligence**.\n\n---\n\n*This RAG memory system transforms AI Khata from a reactive inventory tool into a proactive sales intelligence partner that understands your business.*
